1. PROJECT OVERVIEW

1.1 Project Title

Eduquity Hardware Management System

1.2 Project Description

A comprehensive web-based hardware management system designed for Eduquity, a thought-leader in the Indian assessment industry since 2000. The system manages hardware inventory, employee assignments, serial number tracking, and verification processes for exam duty equipment.

1.3 Key Objectives

  • Track hardware inventory with serial numbers
  • Assign hardware to employees for exam duties
  • Verify serial numbers during assignment and return
  • Provide dual verification (manual and Excel-based)
  • Generate reports and export data
  • Maintain complete audit trail

3. USER ROLES & PERMISSIONS

3.1 User Types

MANAGEREMPLOYEE
  • Full system access
  • Create/Edit hardware
  • Create employees
  • Create projects
  • Create assignments
  • Verify serial entries
  • Return hardware
  • Export reports
  • Limited access
  • View assigned hardware
  • Enter serial numbers
  • Verify received hardware
  • View assignment history
  • Change own password

3.2 First Login Flow

Employee Login → Check is_first_login → (True) → Redirect to Change Password → 
Employee changes password → is_first_login set to False → Full access granted

4. FEATURES & MODULES

4.1 Authentication Module

  • User login with role-based redirection
  • First login password change enforcement
  • Password strength validation
  • Session management
  • Logout functionality

4.2 Dashboard Module

  • Manager Dashboard: Statistics cards, hardware status overview, recent activity feed
  • Employee Dashboard: Current assignments, pending serial entries, completed assignments, exam city display

4.3 Hardware Management Module

  • Add/Edit/Delete hardware items
  • Hardware types management
  • Serial number tracking
  • Status management (Available, Assigned, In Use, Maintenance, Retired)
  • Filter and search functionality
  • Export to Excel

4.4 Employee Management Module

  • Create employee accounts
  • Automatic temporary password generation
  • Email credentials delivery
  • Employee listing with search/filter
  • Delete employee (with active assignment check)

4.5 Assignment Module

  • Create assignments with manual serial entry
  • Select employee and project
  • Specify exam city and return date
  • Real-time serial number verification
  • Multiple hardware items per assignment
  • Return assignment with serial verification

4.6 Verification Module (Dual Verification)

  • Manual Verification: Employee enters serial numbers, Manager verifies each entry
  • Auto Verification (Excel): Employee generates verification Excel, Manager uploads, System auto-verifies matching serials

5. DATABASE SCHEMA

5.1 Models

CustomUser Model:

class CustomUser(AbstractUser):
    USER_TYPE_CHOICES = (
        ('manager', 'Manager'),
        ('employee', 'Employee'),
    )
    user_type = models.CharField(max_length=20, choices=USER_TYPE_CHOICES)
    is_first_login = models.BooleanField(default=True)
    manager = models.ForeignKey('self', on_delete=models.SET_NULL, null=True, blank=True)
    phone = models.CharField(max_length=15, blank=True, null=True)

Hardware Model:

class Hardware(models.Model):
    STATUS_CHOICES = (
        ('available', 'Available'),
        ('assigned', 'Assigned'),
        ('in_use', 'In Use'),
        ('maintenance', 'Maintenance'),
        ('retired', 'Retired'),
    )
    hardware_type = models.ForeignKey(HardwareType, on_delete=models.CASCADE)
    serial_number = models.CharField(max_length=100, unique=True)
    model_name = models.CharField(max_length=100)
    brand = models.CharField(max_length=100, blank=True, null=True)
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default='available')

8. INSTALLATION GUIDE

8.1 Prerequisites

# Install Python 3.11+
python --version

# Install pip
pip --version

8.2 Clone Repository

git clone https://github.com/your-repo/eduquity-hardware.git
cd eduquity-hardware

8.3 Create Virtual Environment

# Windows
python -m venv venv
venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate

8.4 Install Dependencies

pip install -r requirements.txt

requirements.txt:

Django==4.2
mysqlclient==2.2.0
openpyxl==3.1.2
pandas==2.1.0
Pillow==10.0.0
python-decouple==3.8

8.5 Run Migrations

python manage.py makemigrations
python manage.py migrate

8.6 Create Superuser

python manage.py createsuperuser

8.7 Run Development Server

python manage.py runserver
Note: Access the application at http://127.0.0.1:8000/

11. TROUBLESHOOTING GUIDE

Common Issues & Solutions

Issue 1: Database migration errors

Solution:
python manage.py migrate --fake
python manage.py migrate

Issue 2: Email not sending

Check:
- EMAIL_HOST settings in settings.py
- SMTP server credentials
- Firewall settings

Issue 3: File upload fails

Check:
- MAX_UPLOAD_SIZE in settings
- File permissions
- Disk space
